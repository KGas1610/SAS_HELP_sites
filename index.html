SAS Procedures

🏷
PROC FORMAT
Relabel coded values for display without altering raw data
Formatting
What it does
  • Changes how values are displayed, not stored
  • Maps numeric or character codes → readable labels
  • Formats are defined once and recalled in PROC PRINT
  • Apply to a single variable or a range
Use cases
  • Show YES/NO instead of 1/0 for a MARRIED variable
  • Show FEMALE/MALE instead of F/M for GENDER
  • Display meaningful labels in reports & outputs
  • Avoid manual recoding of raw data
Numeric format example
/* Define a format for a numeric variable */
PROC FORMAT DATA=MYDATA;
  VALUE FMTMARRIED  0 = "NO"
                     1 = "YES";
RUN;

/* Apply format in PROC PRINT */
ODS HTML;
PROC PRINT DATA=MYDATA;
  FORMAT MARRIED FMTMARRIED.;
RUN;
ODS HTML CLOSE;
Character format example
/* $ prefix required for character variable formats */
PROC FORMAT DATA=MYDATA;
  VALUE $FMTGENDER  "F" = "FEMALE"
                     "M" = "MALE";
RUN;

ODS HTML;
PROC PRINT DATA=MYDATA;
  FORMAT GENDER $FMTGENDER.;
RUN;
ODS HTML CLOSE;
💡

Remember: numeric format names have no dollar sign; character format names always start with $. Also, when calling the format, always end with a period (FMTMARRIED.).

🖨
PROC PRINT (Modified)
Control the look and content of printed output
Output
DOUBLE
Double-spaces the printed output rows
LABEL
Displays variable labels as column headers
N = "text"
Adds a heading & total count of observations
OBS = "text"
Custom label for the observation number column
ROUND
Rounds numeric values before printing
PROC PRINT with options
ODS HTML;
PROC PRINT DATA=MYDATA
           DOUBLE
           LABEL
           N="Total observations:"
           OBS="Record #"
           ROUND;
RUN;
ODS HTML CLOSE;
📌

Options go in the PROC statement after the data set name, before the semicolon. Reference file: APRINT1.SAS

📤
ODS — Output Delivery System
Export and style SAS output to HTML, PDF, RTF and more
Output
How it works
  • Wrap target code between ODS … ; and ODS … CLOSE;
  • HTML is the default from SAS 9.4 onwards
  • Replace HTML with PDF, RTF, etc. for other formats
Destination options
  • HTML — browser-viewable, default
  • PDF — printable document
  • RTF — Word-compatible rich text
  • LISTING — plain text / classic SAS output
ODS template
/* Wrap any PROC in ODS to enhance its output */
ODS HTML;              /* open HTML destination */

PROC PRINT DATA=MYDATA;
  FORMAT MARRIED FMTMARRIED.;
RUN;

ODS HTML CLOSE;        /* close — must always close! */

/* To produce a PDF instead: */
ODS PDF;
  /* ... proc code ... */
ODS PDF CLOSE;

TITLE & FOOTNOTE Statements
Add descriptive text to the top or bottom of output
Output
TITLE
  • Appears at the top of each output page
  • Supports up to TITLE10 (numbered)
  • Declare without text to delete the title
FOOTNOTE
  • Appears at the bottom of each output page
  • Supports up to FOOTNOTE10 (numbered)
  • Declare without text to delete the footnote
Usage
TITLE "Employee Survey Results 2024";
TITLE2 "Subset: Department A";
FOOTNOTE "Source: HR Database";

PROC PRINT DATA=MYDATA; RUN;

/* Clear titles and footnotes after use */
TITLE;
FOOTNOTE;
💬
Comments in SAS Code
Annotate your programs for clarity and documentation
Syntax
Block comment /* … */
  • Can span multiple lines
  • Useful for commenting out entire sections
  • Cannot be nested inside each other
Statement comment * … ;
  • Starts with * and ends with ;
  • Single line or multi-line
  • Must appear where a SAS statement can go
Both comment styles
/* This is a block comment.
   It can span multiple lines.
   SAS ignores everything inside. */

* This is a statement comment;
* Another statement comment on its own line;

DATA MYDATA;
  /* Inline comment explaining a step */
  SET RAWDATA;
RUN;
🟢

In the SAS editor, all commented text appears in green. Use this as a visual cue that your comment syntax is correct.

Evaluating Quantitative Data

📊
PROC MEANS
Descriptive statistics for quantitative (numeric) variables
Statistics
What it does
  • Evaluates quantitative/numerical data
  • Defines means by groups (BY / CLASS)
  • Helps spot outliers quickly
  • Produces a simple statistical summary report
Default statistics produced
  • N — number of observations
  • MEAN — arithmetic average
  • STD — standard deviation
  • MIN — minimum value
  • MAX — maximum value
Full syntax reference
PROC MEANS DATA=MYDATA  MAXDEC=2
           N MEAN STD MIN MAX MEDIAN RANGE SUM;
  VAR   variable1 variable2;      /* variables to analyse */
  BY    GroupVar;                  /* must be pre-sorted */
  /* OR */
  CLASS GroupVar;                  /* no pre-sort needed */
RUN;
Options (PROC statement)
DATA =
Specifies the input dataset
MAXDEC = n
Number of decimal places in output
N
Count of non-missing observations
NMISS
Count of missing observations
MEAN
Arithmetic average
STD
Standard deviation
MIN / MAX
Minimum and maximum values
RANGE
MAX − MIN
SUM
Total sum of all values
VAR
Variance (STD²)
STDERR
Standard error of the mean
MEDIAN
Middle value (50th percentile)
CLM
Confidence limits for the mean
BY vs CLASS statement
/* ── BY: dataset must be pre-sorted ── */
PROC SORT DATA=MYDATA; BY GENDER; RUN;

PROC MEANS DATA=MYDATA MEAN STD;
  VAR AGE SALARY;
  BY GENDER;         /* one table per GENDER value */
RUN;

/* ── CLASS: no pre-sorting required ── */
PROC MEANS DATA=MYDATA MEAN STD;
  VAR AGE SALARY;
  CLASS GENDER;      /* groups data by GENDER automatically */
RUN;
💡

BY vs CLASS: Use BY when your data is already sorted (or you PROC SORT first). Use CLASS for convenience — it handles grouping without sorting, and produces a single consolidated table. Reference files: AMEANS.1.SAS, AMEANS.2.SAS

📈
PROC UNIVARIATE
Advanced distribution analysis, normality testing and graphical plots
Distribution
What it does
  • Tests whether data is normally distributed
  • Identifies outliers in the data
  • Produces histograms, box plots, probability plots
  • Compares distributions across groups
Key output metrics
  • Skewness — symmetry (should be ≈ 0)
  • Kurtosis — shape (should be = 0)
  • Positive kurtosis → more peaked than normal
  • Negative kurtosis → flatter than normal
Full syntax reference
PROC UNIVARIATE DATA=MYDATA  NORMAL  PLOT;
  VAR       variable1 variable2;
  HISTOGRAM variable1 / NORMAL(COLOR=RED W=2);
  QQPLOT    variable1;
  PROBPLOT  variable1;
  INSET     MEAN STD / FORMAT=(4.0);
RUN;
PROC statement options
DATA =
Input dataset to analyse
NORMAL
Adds normality test table (Shapiro-Wilk etc.)
PLOT
Produces stem-and-leaf, box & probability plots
TRIMMED
Trims top & bottom before computing the mean
CIBASIC
Confidence intervals for mean, std and var
ALPHA =
Sets alpha level for the intervals
MU0 =
Parameter for a t-test of the mean
Statement options
VAR
Variable(s) to analyse
HISTOGRAM / NORMAL
Bar chart of distribution with optional bell curve overlay
HISTOGRAM / COLOR
Colour of the superimposed normal curve
HISTOGRAM / W =
Line width of the normal curve
QQPLOT
Higher-quality normal probability plot
PROBPLOT
Probability plot for checking normality
INSET
Displays stats inside the graph. FORMAT = (4.0) controls decimal display
Practical example with histogram
/* Test normality and visualise distribution of AGE */
PROC UNIVARIATE DATA=MYDATA NORMAL;
  VAR AGE;
  HISTOGRAM AGE / NORMAL(COLOR=BLUE W=2);
  INSET MEAN STD SKEWNESS KURTOSIS / FORMAT=(6.2);
  QQPLOT AGE;
RUN;
💡

Interpreting output: If Skewness ≈ 0 and Kurtosis ≈ 0, the data is approximately normal. The normality test p-value (from the NORMAL option) should be > 0.05 to confirm normality. Reference files: AUNI1.SAS, AUNI2.SAS, AUNI4.SAS

📌

PROC MEANS vs PROC UNIVARIATE: Use PROC MEANS for a quick summary table of common stats. Use PROC UNIVARIATE when you need normality tests, detailed distribution plots, or outlier identification.
